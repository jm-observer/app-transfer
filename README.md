# app-transfer Example Program

## Overview

`example` is a small Rust utility that demonstrates basic asynchronous I/O handling with Tokio. It performs two concurrent tasks:

1. **Echo with suffix** – Reads lines from **standard input**, appends a configurable suffix (`_suffix`), and writes the result to **standard output**.
2. **Periodic stderr output** – In a background task, every 5 seconds it writes an increasing counter to **standard error**. This shows how to spawn a background asynchronous task alongside the main I/O loop.

The program is deliberately simple so it can be used as a template for building more complex command‑line tools that need both foreground and background asynchronous work.

## Build Requirements

- **Rust** (stable 1.70+ recommended)
- **Cargo** – the Rust package manager (installed with Rustup)
- **Tokio** – used for async runtime (already declared in `Cargo.toml`)

## Building the Binary

```bash
# Build in debug mode (fast, includes debug symbols)
cargo build

# Or build a release binary (optimised)
cargo build --release
```

The resulting executable will be placed under `target/debug/example.exe` (debug) or `target/release/example.exe` (release).

## Running the Program

```bash
# Simple usage: pipe a string into the program
echo "hello" | ./target/debug/example.exe
```

**Expected output**

* **stdout** – `hello_suffix`
* **stderr** – a counter that starts at `0` and increments every 5 seconds (e.g., `0`, `1`, `2`, …) while the program is running.

### Interactive mode
You can also run the binary directly and type lines manually:

```bash
./target/debug/example.exe
```
Enter lines; each line you type will be echoed with the `_suffix` appended.

## How It Works

- The `main` function spawns a Tokio task (`task::spawn`) that runs an infinite loop with a 5‑second interval (`time::interval`). This task writes the current counter value to `eprintln!`, which goes to stderr.
- Meanwhile, the main task reads from `stdin` line‑by‑line using a buffered reader (`io::BufReader::new(io::stdin())`). Each line is trimmed of its trailing newline, suffixed, and written to `stdout`.
- The program exits cleanly when EOF is encountered on stdin (e.g., when piping a file or using `Ctrl‑D` on Unix‑like shells).
